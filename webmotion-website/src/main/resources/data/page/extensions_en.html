Extensions

Principle

Webmotion is part of a component-oriented approach, with its extension system. The extensions enable the code reuse from one application to another, and even within a single application.

Reminder, the benefits of the use of extensions in your project are significant

  • Reuse : you can pool business components between different projects.
  • Structuring : the extensions enable to cut out a project in functional or technical..
  • Modularity : during deployment, the extensions can be determined dynamically.

If you want to know more about the realization and configuration of extensions, please refer to the Mapping - Extensions section.

Wikimotion

Wikimotion is a wiki engine which can be used both standalone or as an extension. For example, the website WebMotion uses the wiki as an extension. A default configuration file wikimotion-config.properties is provided, but you can modify it to configure and customize your wiki.

It offers the following features:

  • Flat files saving (no database needed)
  • Pages written in HTML, RST or LaTeX
  • Basic HTTP authentication support
  • File upload
  • Site map
  • Media map

Spring

WebMotion offers a dependency injection extension based on Spring. It enables you to manage the controllers (WebMotionController) and the WebMotionHandlers in Spring. This extension also enables to easily use the Spring scopes (request/session/singleton/prototype) for your objects.

To use it, you need to get the jar and modify the [config] section of your mapping file as follows: 

[config]
server.main.handler.class=org.debux.webmotion.spring.SpringMainHandler

The configuration files are automatically loaded from the WEB-INF directory. Their name must starts with applicationContext.

applicationContext.xml file example: 

<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
                           http://www.springframework.org/schema/beans/spring-beans-3.0.xsd
			   http://www.springframework.org/schema/context
                           http://www.springframework.org/schema/context/spring-context-3.0.xsd">

    <context:annotation-config/>

    <bean ... />

</beans>

You need to use the dependency injection to make calls between actions instead of renderAction: 

public class Spring extends WebMotionController {

    @Inject
    protected Action action;

    public Render run() {
        return action.exec();
    }

}
